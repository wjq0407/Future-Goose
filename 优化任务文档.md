## 一、项目诊断分析

### 1.1 产品定位回顾
- **目标用户**: 在校大学生（大一至研究生）
- **核心价值**: AI驱动的职业规划 + 腾讯生态对接
- **品牌标识**: "鹅"字Logo（腾讯企鹅隐喻），名称巧妙融合"未来"+"鹅厂"

### 1.2 现有视觉问题诊断

| 维度 | 现状 | 问题 |
|------|------|------|
| **品牌融合度** | Logo用了"鹅"字，配色以通用蓝为主 | 腾讯元素停留在表面，缺乏深度文化融入 |
| **视觉独特性** | 使用常见的渐变+玻璃拟态 | 设计趋同，缺乏辨识度，没有"鹅厂基因" |
| **色彩系统** | 蓝色系 + 紫色渐变，通用性强 | 没有采用腾讯品牌色 `#0052D9`，缺少鹅厂专属色板 |
| **IP运用** | 仅文字"鹅" | 缺少企鹅IP元素的现代化表达 |
| **文化表达** | 鹅厂专区有文字内容 | 视觉层面没有体现腾讯"自树而生"、"科技向善"等理念 |
| **青年审美** | 常规卡片+渐变 | 缺少当代年轻人喜爱的微交互、游戏化元素 |

### 1.3 工程化现状
- **性能**: 已有骨架屏、懒加载、Web Vitals监控，基础良好
- **安全**: API Key存储在localStorage，需加强
- **可扩展**: Tailwind配置已扩展但缺少系统性设计令牌
- **暗色模式**: 配置了 `darkMode: "class"` 但未充分实现

## 二、设计哲学：腾讯基因 × 青年文化

### 2.1 核心设计理念

```
"自树而生" —— 扎根成长，向上辐射
    ↓
以腾讯蓝为根，用户成长路径为枝干
每个阶段的成长，都是向上生长的新枝叶
```

**关键词**: 
- 🌱 **生长感** - 界面元素呈现向上生长的动势
- 🐧 **鹅厂DNA** - 企鹅IP的现代扁平化演绎
- 🔵 **科技蓝** - 腾讯标准色贯穿核心交互
- ✨ **青春活力** - 年轻、活力、不呆板
- 🎯 **陪伴感** - 温暖、亲切、像学长一样

### 2.2 腾讯文化元素映射表

| 腾讯文化 | 视觉转译 | 应用场景 |
|----------|---------|---------|
| **自树而生** | 向上生长的渐变、树枝状时间线 | 成长规划页、进度动画 |
| **斜8度切割** | 8度倾斜的装饰元素、按钮微倾 | 卡片装饰角、分割线 |
| **瑞雪文化** | 纯白+微蓝的简洁界面 | 对话页面、表单页面 |
| **科技向善** | 温暖的绿色辅助色 | 完成状态、成就徽章 |
| **用户为本** | 大圆角、圆润图标 | 全局交互元素 |
| **8周年品牌色** | 蓝→紫→粉渐变 | Hero区域、强调元素 |
| **PUPU企鹅IP** | 极简企鹅几何图形 | Logo装饰、空状态插画 |
| **T族/P族双通道** | 双轨道进度可视化 | 成长路径展示 |

### 2.3 当代青年审美融入

| 趋势 | 融入方式 | 克制原则 |
|------|---------|---------|
| **玻璃拟态2.0** | 更自然的毛玻璃+微渐变背景 | 仅用于卡片，不过度 |
| **微交互** | 按钮点击反馈、进度填充动画 | 时长≤300ms，不打扰 |
| **游戏化** | 成就徽章、等级标识 | 仅在成长/测评模块 |
| **莫兰迪色系** | 降低辅助色饱和度 | 保持腾讯蓝纯度 |
| **超大圆角** | R≥16px | 统一圆角系统 |
| **暗色模式** | 非纯黑 #121212 | 降低蓝光，护眼 |

#### 任务 4.1：前端安全加固
**优先级**: P0 🔴  
**工作量**: 中  
**影响范围**: 全局

**具体任务**:

   - 限制脚本来源为 `self` + 必要的CDN
   - 禁止 `eval()` 使用

3. **XSS防护**:
   - 审查所有 `innerHTML` / `dangerouslySetInnerHTML` 使用
   - Markdown渲染确保使用DOMPurify过滤
   - 用户输入输出时做HTML实体编码

4. **依赖安全**:
   - 运行 `npm audit` 并修复高危漏洞
   - 添加 `.npmrc` 配置自动审计
   - 关键依赖锁定版本号

**验收标准**:

## 五、技术约束与最佳实践

### 5.1 性能红线
- 首屏加载时间 < 2.5s
- 单个动画耗时 ≤ 400ms
- 页面JS体积 < 300KB (gzip)
- 图片懒加载覆盖率 100%
- Lighthouse Performance ≥ 90

### 5.2 安全红线
- API Key必须加密存储
- 所有用户输入必须过滤
- CSP策略必须启用
- 依赖漏洞必须修复高危项

### 5.3 可访问性红线
- 色彩对比度 ≥ 4.5:1
- 键盘可操作所有功能
- 屏幕阅读器兼容

### 5.4 代码规范
- TypeScript strict模式
- ESLint + Prettier
- 组件Props必须有类型定义
- 关键组件必须有注释

### 5.5 动画规范
- 优先使用CSS动画，而非JS
- 动画属性仅限于 `transform` 和 `opacity`
- 必须支持 `prefers-reduced-motion`
- 动效时长 150ms-400ms
- 缓动函数统一使用 `cubic-bezier(0.4, 0, 0.2, 1)`
